ENTRADA 1
---------
FECHA: Abril 2026
SECCIÓN: Estructura base del proyecto
PROMPT: "Crea la estructura base de un proyecto web JavaScript siguiendo el 
         patrón MVC para una web de piscina de colegio. Necesito: index.html,
         components.js, app.js, Rules.md, SPEC.md"
RESULTADO: Creación de la estructura inicial del proyecto con todos los 
            archivos base comentados y funcionando
NOTAS: Proyecto iniciado con éxito. Funciones puras en components.js, 
       manipulación del DOM solo en app.js

ENTRADA 2
---------
FECHA: 23 Abril 2026
SECCIÓN: Componente Horarios (Persona 1)
PROMPT: "necesito que mejores los horarios, que sean todos de tarde, la piscina 
         abre a partir de las 17:00 y el horario ha de ser diferente para mayores 
         y pequeños, el diseño ha de ser bien estructurado y minimalista, con 
         tonos amarillos y morados"
RESULTADO: Se actualizó Horarios.css con paleta amarillo/morado. Se intentó 
            reestructurar Horarios.js con tarjetas separadas para mayores y 
            pequeños, pero el usuario revirtió los cambios del JS manteniendo 
            la estructura original de tabla única.
NOTAS: El usuario prefiere mantener la estructura de tabla original del JS.

ENTRADA 3
---------
FECHA: 23 Abril 2026
SECCIÓN: Diseño global de la página y funcionalidad
PROMPT: "Mejora y arregla los errores en la página sin cambiar toda la estructura, 
         ha de ser el diseño minimalista con tonos morados y amarillos. haz que la 
         página sea funcional, además guarda los prompts que te hecho en 
         mensajes_prompts.txt"
RESULTADO: Se actualizaron los 7 archivos CSS (Header, Nav, InfoSection, Horarios, 
            Precios, Actividades, Footer) con paleta morado (#7B2D8E) y amarillo 
            (#F5C518). Se corrigió HTML inválido (<main> anidado en InfoSection). 
            Se añadieron ids a las secciones y se implementó navegación funcional 
            con scroll suave en app.js. Se actualizó index.html con SEO y 
            smooth scrolling.
NOTAS: Todos los componentes mantienen su estructura JS original. Solo se 
       cambiaron estilos y se añadió funcionalidad de navegación.

ENTRADA 4
---------
FECHA: 28 Abril 2026
SECCIÓN: Actividades (Persona 3 - Aitor)
PROMPT: "Añade el apartado 'Más información' de cada actividad en la carpeta

RESULTADO: Se actualizó Actividades.js con un nuevo campo 'masInfo' para cada

NOTAS: Se mejoró la información detallada de cada actividad para proporcionar

ENTRADA 5
---------
FECHA: 28 Abril 2026
SECCIÓN: Inscripción (Persona 3 - Aitor)
PROMPT: "He visto que has añadido un texto 'Inscribirse ahora'. Haz también ese 
         apartado. Y añade este prompt a mensajes prompts.txt (hazlo con todo 
         lo que te diga a partir de ahora)"
RESULTADO: Creación del componente de Inscripción (Inscripcion.js/css) con un 
            formulario modal interactivo. Se implementó la lógica en app.js 
            para abrir el modal, autoseleccionar la actividad y gestionar 
            el envío simulado del formulario con mensaje de éxito.
NOTAS: El sistema de inscripción es ahora funcional visualmente y mejora la 
        interactividad de la sección de actividades.

ENTRADA 6
---------
FECHA: 28 Abril 2026
SECCIÓN: Exportación Excel (Persona 3 - Aitor)
PROMPT: "Ahora quiero que meta cada inscripción en un archivo .xls (Excel)"
RESULTADO: Integración de la librería SheetJS vía CDN en index.html. Se añadió 
            la función exportToExcel en app.js que captura los datos del 
            formulario y genera automáticamente la descarga de un archivo 
            Excel (.xls) al confirmar la inscripción.
NOTAS: Se ha habilitado la descarga local del archivo para permitir la gestión 
        administrativa de las inscripciones sin necesidad de base de datos.

CONVENCIONES DEL PROYECTO
================================================================================

✅ PERMITIDO:
   - Funciones render* que devuelven HTML String
   - Template literals para construir HTML
   - Event listeners en app.js
   - querySelector, getElementById en app.js
   - innerHTML en app.js

❌ PROHIBIDO:
   - appendChild en components.js
   - querySelector en components.js
   - innerHTML en components.js
   - Event listeners en components.js
   - Acceso directo al DOM fuera de app.js

NOTAS DE DESARROLLO
================================================================================

El patrón MVC elegido es una versión simplificada adecuada para:
- Proyectos medianos
- Aprendizaje de patrones de arquitectura
- Código mantenible y escalable
- Sin dependencias externas

Ventajas de esta aproximación:
- Código limpio y modular
- Fácil de testear (components.js con funciones puras)
- Separación de responsabilidades
- Flexible para agregar más funcionalidades
